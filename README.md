# 오목 게임 규칙 및 요구사항

## 게임 규칙
- **보드 크기**: 표준 15x15 격자를 사용하며, 좌표 표기(예: A1~O15)를 지원합니다.
- **돌 배치 순서**: 흑이 먼저 두고 이후 흑-백이 번갈아 1수씩 착수합니다.
- **승리 조건**: 가로나 세로, 또는 대각선으로 연속 5개의 돌을 먼저 완성한 플레이어가 승리합니다. 백은 6목 이상도 승리로 인정되지만, 흑은 금수 규칙을 따릅니다.
- **금수 규칙(흑 한정)**:
  - 장목: 6목 이상을 만드는 수는 금지됩니다.
  - 삼삼: 동시에 두 개 이상의 열린 삼(333)을 만드는 수는 금지됩니다.
  - 사사: 동시에 두 개 이상의 열린 사(444)을 만드는 수는 금지됩니다.
- **무승부 처리**: 모든 교차점이 채워질 때까지 승패가 나지 않으면 무승부로 처리합니다.

## 기능 요구사항
- **게임 모드**
  - 싱글 플레이: 기본 AI(무작위 또는 간단한 휴리스틱)와 대전하며, 난이도 선택(예: 쉬움/보통)을 제공합니다.
  - 로컬 멀티 플레이: 하나의 디바이스에서 번갈아 착수하는 패스-앤-플레이 방식을 지원합니다.
  - (선택) 온라인 멀티 플레이: 추후 확장을 고려해 사용자 매칭 및 실시간 동기화를 염두에 둔 구조를 설계합니다.
- **UI 방식**
  - 1차 목표는 터미널/콘솔 기반 UI로, 좌표 입력과 직관적인 보드 렌더링(격자, 최근 착수 표시)을 제공합니다.
  - 추가적으로 GUI(예: 웹 또는 데스크톱)로 확장할 경우, 클릭 기반 착수, 되돌리기 요청, 승패 알림 모달을 포함합니다.
- **품질 및 편의 기능**
  - 착수 취소(Undo) 기능은 싱글 플레이에서만 허용하며 최대 N회까지 제한합니다.
  - 게임 기록(수순, 승패, 시간)을 저장하고 재생할 수 있는 로그 기능을 제공합니다.
  - 금수 감지를 위한 실시간 경고 또는 착수 거부 메시지를 제공합니다.

이 문서는 오목 게임을 설계하고 구현하는 과정에서의 기준점으로 활용됩니다.
